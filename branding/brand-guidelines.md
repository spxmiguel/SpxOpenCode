# SpxOpenCode Brand Guidelines

## Name

**SpxOpenCode** — always written as one word, capital S, capital O, capital C.

- ✅ SpxOpenCode
- ❌ SPXOpenCode / spx-opencode / SPX OpenCode / Spx Open Code

Short form: **SPX** (all caps) — used in logo icon, status bar, and compact contexts.

## Logo

Located at `branding/logo.svg` and `branding/icon.svg`.

**Do:**
- Use on dark backgrounds (#0D1117 or equivalent)
- Maintain minimum clear space of 16px around logo
- Scale proportionally only

**Don't:**
- Change colors
- Add drop shadows or effects
- Place on light backgrounds without a dark container
- Stretch or distort

### Minimum sizes

| Context | Minimum width |
|---------|---------------|
| Full logo | 120px |
| Icon only | 24px |
| Banner | 800px wide |

## Typography

All brand typography uses monospace fonts to reinforce terminal/developer identity.

| Usage | Font | Fallback |
|-------|------|---------|
| Logo / wordmark | JetBrains Mono | Fira Code, Courier New, monospace |
| UI labels | JetBrains Mono | monospace |
| Body text | Inter | Helvetica Neue, sans-serif |
| Code | JetBrains Mono | monospace |

## Voice and Tone

SpxOpenCode communication is:

- **Direct** — no filler words, no marketing fluff
- **Technical** — written for developers, not managers
- **Honest** — never overpromise; "planned" means planned, not shipped
- **Minimal** — one sentence when one sentence is enough

Avoid:
- Exclamation marks in docs or release notes
- Phrases like "powerful", "revolutionary", "game-changing"
- Emoji in technical documentation (acceptable in changelogs for scan-ability)

## What SpxOpenCode is NOT

- Not a company product
- Not a SaaS
- Not a competitor to OpenCode — it's built on top of it
- Not an AI wrapper with its own model routing by default

Keep brand positioning accurate. SpxOpenCode is a fork with productivity plugins, not a standalone AI tool.
